客户端配置¶

更新客户端配置¶

Certain client configurations can be applied at runtime. To check if a configuration option can be applied (taken into affect by a client) at runtime, use the config help command:

ceph config help debug_client
 debug_client - Debug level for client
 (str, advanced)                                                                                                                      Default: 0/5
 Can update at runtime: true

 The value takes the form 'N' or 'N/M' where N and M are values between 0 and 99.  N is the debug level to log (all values below this are included), and M is the level to gather and buffer in memory.  In the event of a crash, the most recent items <= M are dumped to the log file.

config help tells if a given configuration can be applied at runtime along with the defaults and a description of the configuration option.

To update a configuration option at runtime, use the config set command:

ceph config set client debug_client 20/20

Note that this changes a given configuration for all clients.

To check configured options use the config get command:

ceph config get client
 WHO    MASK LEVEL    OPTION                    VALUE     RO
 client      advanced debug_client              20/20
 global      advanced osd_pool_default_min_size 1
 global      advanced osd_pool_default_size     3

客户端配置参考¶

client_acl_type¶

设置 ACL 类型。现在还只能设置为 "posix_acl" 表示启用 POSIX ACL ，或者可设置为空字符串。此选项只有在 fuse_default_permissions 被设置为 false 时有效。

type

str

client_cache_mid¶

设置客户端缓存中点。此中点把最近用过的列表分割为热列表和暖列表。

type

float

default

0.75

client_cache_size¶

设置客户端保留在元数据缓存中的 inode 数量。

type

size

default

16Ki

client_caps_release_delay¶

设置释放能力的延时，单位为秒。 这个延时控制着客户端不再需要能力时， 再等多少秒就释放， 以便其它需要这些能力的用户空间操作使用。

type

int

default

5

client_debug_force_sync_read¶

设置为 true 时，客户端会跳过本地页缓存、 直接从 OSD 读取数据。

type

bool

default

false

client_dirsize_rbytes¶

此选项可以启用一个 CephFS 功能， 就是用 stat 数据结构的 st_size 字段存储递归目录的尺寸 （此目录内的文件及其子孙们占用的空间）。

type

bool

default

true

client_max_inline_size¶

控制内联数据的最大尺寸，小于此尺寸就存储到文件的索引节点内， 超过则存到单独的 RADOS 对象内。 本选项只有设置了 MDS 图的 inline_data 标志时有效。

type

size

default

4Ki

client_metadata¶

客户端向各 MDS 发送元数据时，除了自动生成的版本号、主机名等信息， 还可以附加逗号分隔的字符串作为附加元数据。

type

str

client_mount_gid¶

设置 CephFS 挂载后的组 ID 。

type

int

default

-1

client_mount_timeout¶

设置挂载 CephFS 时的超时时间，单位为秒。

type

float

default

5 minutes

client_mount_uid¶

Set the user ID of CephFS mount.

type

int

default

-1

client_mountpoint¶

指定要挂载的 CephFS 文件系统目录。 此选项的作用类似 ceph-fuse 命令的 -r 选项。

type

str

default

/

client_oc¶

enable object caching

type

bool

default

true

client_oc_max_dirty¶

设置对象缓存的脏数据上限，单位为字节。

type

size

default

100Mi

client_oc_max_dirty_age¶

用于设置回写之前，脏数据在对象缓存中的最大存留时间， 单位为秒。

type

float

default

5.0

client_oc_max_objects¶

设置对象缓存允许的最大对象数。

type

int

default

1000

client_oc_size¶

设置客户端可缓存的数据上限，单位为字节。

type

size

default

200Mi

client_oc_target_dirty¶

设置认定为脏数据的目标尺寸。我们建议这个数字尽量小些。

type

size

default

8Mi

client_permissions¶

检查所有 I/O 操作的客户端权限。

type

bool

default

true

client_quota_df¶

让 statfs 操作报告根目录的配额。

type

bool

default

true

client_readahead_max_bytes¶

设置客户端预读数据的最大尺寸， 单位为字节。 本选项可被 client_readahead_max_periods 覆盖。

type

size

default

0B

client_readahead_max_periods¶

设置客户端预读的文件布局分片最大数量 （对象尺寸 * 条带数量）。 本选项会覆盖 client_readahead_max_bytes 选项。

type

int

default

4

client_readahead_min¶

设置客户端预读的最小尺寸，单位为字节。

type

size

default

128Ki

client_reconnect_stale¶

是否自动重连过期的会话。

type

bool

default

false

client_snapdir¶

设置快照目录名。

type

str

default

.snap

client_tick_interval¶

设置更新能力及维持其它信息的间隔时长，单位为秒。

type

secs

default

1

client_use_random_mds¶

为各个请求随机选取 MDS 。

type

bool

default

false

fuse_default_permissions¶

设置为 false 时， ceph-fuse 工具会用自己的权限验证机制， 而非依靠 FUSE 的强制权限。 启用 POSIX ACL 需把此选项设置为 false 、 同时设置 client acl type=posix_acl 。

type

bool

default

false

fuse_max_write¶

设置单次写入操作的最大字节数。 值为 0 表示维持不变，还是用 FUSE 的默认值 128kB 。

type

size

default

0B

fuse_disable_pagecache¶

If set to true, kernel page cache is disabled for ceph-fuse mounts. When multiple clients read/write to a file at the same time, readers may get stale data from page cache. Due to limitations of FUSE, ceph-fuse can’t disable page cache dynamically.

type

bool

default

false
