MDS 配置参考

mds_cache_mid

把新条目插入缓存 LRU 时的插入点（从顶端）。

type

float

default

0.7

mds_dir_max_commit_size

一个目录更新超过多大时要拆分为较小的事务（ MB ）。

type

int

default

10

mds_dir_max_entries

目录里的最大条数，超过时再新建就会被拒绝， 遇到 ENOSPC 错误。

type

uint

default

0

mds_decay_halflife

用于均衡的各个目录、其热度计数器的衰减速度。

type

float

default

5.0

mds_beacon_interval

MDS 和 监视器们之间发送信标消息的间隔秒数。

type

float

default

4.0

mds_beacon_grace

多久没收到信标消息 Ceph 就认为 MDS 落后了 （并可能替换它）。

type

float

default

15.0

mon_mds_blocklist_interval

OSD 运行图里的 MDS 失败后，把它留在黑名单里的时间。 注意，此选项控制着失败的 OSD 在 OSDMap 黑名单里呆多长时间， 它对管理员手动加进黑名单的东西没影响。 例如， ceph osd blocklist add 仍将遵循默认的黑名单时间。

type

float

default

1 day

min

1_hr

mds_reconnect_timeout

MDS 处于重建连接恢复状态时，客户端重连的超时时间， 秒数。

type

float

default

45.0

mds_tick_interval

MDS 执行内部周期性任务的间隔。

type

float

default

5.0

mds_dirstat_min_interval

为避免传播沿着目录树上行的 stat 递归操作， 尝试的最小间隔秒数。

type

float

default

1.0

mds_scatter_nudge_interval

dirstat 变更传播多快。

type

float

default

5.0

mds_client_prealloc_inos

为每个客户端会话预分配索引节点号的个数。

type

int

default

1000

mds_early_reply

请求成功提交到日志前 MDS 是否允许客户端先看到请求结果。

type

bool

default

true

mds_default_dir_hash

用于哈希文件在目录碎片中分布情况的函数。

type

int

default

2

mds_log_skip_corrupt_events

在重放日志时， MDS 是否应跳过损坏的日志事件。

type

bool

default

false

mds_bal_sample_interval

对目录热度取样的频率 （作为分片决策的依据）。

type

float

default

3.0

mds_bal_replicate_threshold

达到多大热度时 Ceph 就把元数据复制到其它节点。

type

float

default

8000.0

mds_bal_unreplicate_threshold

热度低到多少时 Ceph 就不再把元数据复制到其它节点。

type

float

default

0.0

mds_bal_split_size

目录尺寸大到多少时 MDS 就把目录分片拆分成更小的片段。

type

int

default

10000

mds_bal_split_rd

目录的最大读取热度达到多大时 Ceph 将拆分此片段。

type

float

default

25000.0

mds_bal_split_wr

目录的最大写热度达到多大时 Ceph 将 拆分此片段。

type

float

default

10000.0

mds_bal_split_bits

把一个目录片段再分割成多大。

type

int

default

3

allowed range

[1, 24]

mds_bal_merge_size

目录尺寸小到多少时 Ceph 就尝试合并邻近的目录片段。

type

int

default

50

mds_bal_interval

MDS 服务器之间交替负荷的频率，秒。

type

int

default

10

mds_bal_fragment_interval

一个片段可以被拆分或合并，在执行分片变更前延迟的时间， 单位为秒。

type

int

default

5

mds_bal_fragment_fast_factor

分片的尺寸超过拆分尺寸阈值达到多少比例就立即执行拆分（跳过 fragment interval 配置的延时）。

type

float

default

1.5

mds_bal_fragment_size_max

一个片段的最大尺寸， 要加入新条目时会收到 ENOSPC 拒绝代码。

type

int

default

100000

mds_bal_idle_threshold

热度低于此值时 Ceph 就把子树 迁移回它的父节点。

type

float

default

0.0

mds_bal_max

均衡器迭代到这么多次时 Ceph 就停止。 （仅适用于测试）

type

int

default

-1

mds_bal_max_until

均衡器运行多久就停止，秒数。 （仅适用于测试）

type

int

default

-1

mds_bal_mode

计算 MDS 负载的方法。

  • 0 = 混合；

  • 1 = 请求速率和延时；

  • 2 = CPU 负载。

type

int

default

0

mds_bal_min_rebalance

子树热度最小多少时开始迁移。

type

float

default

0.1

mds_bal_overload_epochs

The number of epochs the overload lasts before Ceph migrates, setting it to a higher value can avoid frequent migrations caused by load fluctuations.

type

int

default

2

mds_bal_min_start

子树热度最小多少时 Ceph 才去搜索子树。

type

float

default

0.2

mds_bal_need_min

允许接受目标子树片段的最小尺寸。

type

float

default

0.8

mds_bal_need_max

允许接受目标子树片段的最大尺寸。

type

float

default

1.2

mds_bal_midchunk

尺寸大于目标子树的子树， Ceph 将迁移它。

type

float

default

0.3

mds_bal_minchunk

尺寸小于目标子树的子树， Ceph 将忽略它。

type

float

default

0.001

mds_replay_interval

MDS 处于 standby-replay 模式（热备）时的 日志滚动间隔。

type

float

default

1.0

mds_shutdown_check

MDS 关闭期间缓存更新间隔。

type

int

default

0

mds_thrash_exports

Ceph 会在节点间随机地导出子树。（仅用于测试）

type

int

default

0

mds_thrash_fragments

Ceph 会随机地分片或合并目录。

type

int

default

0

mds_dump_cache_on_map

Ceph 会把各 MDSMap 的 MDS 缓存内容转储到一文件。

type

bool

default

false

mds_dump_cache_after_rejoin

Ceph 重新加入缓存（恢复期间）后 会把 MDS 缓存内容转储到一文件。

type

bool

default

false

type

bool

default

false

type

bool

default

false

type

bool

default

false

bool

default

false

bool

default

false

mds_symlink_recovery

mds_extraordinary_events_dump_interval

Interval in seconds for dumping the recent in-memory logs when there is an extra-ordinary event. The default is 0 (disabled). The log level should be < 10 and the gather level should be >=10 in debug_mds for enabling this option.

type

secs

default

0

allowed range

[0, 60]
