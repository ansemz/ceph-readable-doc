常规配置参考

admin_socket

在一个守护进程上执行管理命令的套接字， 与 Ceph 监视器法定人数是否已建立无关。

type

str

default

/var/run/ceph/$cluster-$name.asok 。

pid_file

mon 、 osd 、 mds 将把它们的 PID 写入此文件， 其名为 /var/run/$cluster/$type.$id.pid 。 例如，名为 ceph 的集群，其 ID 为 a 的 mon 进程 将创建 /var/run/ceph/mon.a.pid 。 如果是正常停止的， pid file 就会被守护进程删除； 如果进程未进入后台运行（即启动时加了 -f 或 -d 选项）， 它就不会创建 pid file 。

type

str

chdir

Ceph 进程一旦启动、运行就进入这个目录， 默认推荐 / 。

type

str

see also

daemonize

fatal_signal_handlers

如果设置了，将安装 SEGV, ABRT, BUS, ILL, FPE, XCPU, XFSZ, SYS 的 信号处理器，用于产生有用的日志信息。

type

bool

default

true

max_open_files

如果设置了， Ceph 存储集群启动时 会设置操作系统级的最大打开 FD 数量， 即文件描述符最大允许值， 大小合适的值可防止 Ceph 守护进程们耗尽文件描述符。

类型

64-bit Integer

是否必需

No

默认值

0
