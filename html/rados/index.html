Ceph 存储集群¶

所有 Ceph 部署都始于 Ceph 存储集群。基于 RADOS 的 Ceph 存储集群包括几类守护进程：

  1. Ceph OSD 守护进程（ OSD ）把存储节点上的数据存储为对象；

  2. Ceph 监视器（ MON ）维护集群运行图的主副本；

  3. Ceph 管理器 守护进程

一个 Ceph 集群可以包含数千个存储节点，最简系统至少需要一个监视器和两个 OSD 守护进程才能做到数据复制。

Ceph 文件系统、 Ceph 对象存储、和 Ceph 块设备从 Ceph 存储集群读出和写入数据。

Config and Deploy

Ceph 存储集群有少量配置选项是必要的，但大多数都有默认值。 典型部署是通过部署工具定义集群、并启动监视器的。 详情见Cephadm。
